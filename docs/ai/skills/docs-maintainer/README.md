# Docs Maintainer Skill Spec

The docs maintainer skill should keep HealthBytes documentation current without creating duplicate sources of truth.

Trigger examples:

- Update README.
- Update roadmap.
- Document env vars.
- Update architecture.
- Add weekly report.
- Move docs to legacy.
- Record AI inspection.

Required behavior:

- Read `.cursorrules`.
- Read `.agents/router.json`.
- Check `.agents/agents/tasks.json`.
- Find the correct SSOT.
- Update SSOT first.
- Update navigation links second.
- Never create root docs without approval.
- Preserve legacy docs until migration is approved.
